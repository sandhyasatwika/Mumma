def fahrenheit_to_celsius(fahrenheit):
    """Convert Fahrenheit to Celsius"""

        f = float(input("Enter temperature in Fahrenheit: "))
        print(f"{f}F = {fahrenheit_to_celsius(f):.2f}C")

Excellent question switching to (or creating) a feature branch in Git is one of the most common and useful workflows when developing code especially when collaborating via GitHub.

Heres exactly how to do it

Step-by-Step: Move to a Feature Branch

1. Check what branch youre currently on

bash
git branch

This will list all branches and show the current one with a * (for example, * main).

2. Create a new feature branch (if it doesnt exist yet)

bash
git checkout -b feature/my-new-feature

This command creates and switches to a new branch called feature/my-new-feature.

Common naming pattern:
feature/, bugfix/, or hotfix/ followed by a short description, like:

pgsql
feature/login-page
feature/add-search-bar
bugfix/fix-null-pointer

3. Switch to an existing feature branch

If the branch already exists (either locally or on GitHub):

bash
git checkout feature/my-new-feature

If it exists only on GitHub (remote), first fetch it:

bash
git fetch origin
git checkout feature/my-new-feature

4. Push your new branch to GitHub (if newly created)

bash
git push -u origin feature/my-new-feature

This uploads the branch to GitHub and sets it as your default upstream branch.

5. Verify

Run:

bash
git branch

You should see:

arduino
* feature/my-new-feature
  main

Example Workflow

bash
# Start from main branch
git checkout main

# Update your local main
git pull origin main

# Create & move to new branch
git checkout -b feature/add-login-page

# After making changes
git add .
git commit -m "Add login page UI"

# Push to GitHub
git push -u origin feature/add-login-page
